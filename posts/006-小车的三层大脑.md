# 给AI做身体 | 小车的三层大脑

我在给一辆树莓派小车装 AI 大脑。折腾到现在，整个系统分三层：手机负责听和说，家里的电脑负责转发消息，小车负责想。三层各干各的，全程只传文字，不传音频。

为什么是三层。一开始想得简单，手机录音发给小车，小车跑语音识别再跑大模型。但树莓派算力不够，连语音识别都跑不动，别说大模型了。试过把语音识别放到家里的电脑上，手机录音传过去，转成文字再发给小车。能用，但链路太长，延迟三四秒，而且家里电脑必须一直开着。

最后的方案是让手机自己干语音识别和语音合成。iPhone 系统自带的语音识别就能做实时转文字，语音合成也是系统级的，不需要联网，不需要额外的服务器。手机把录音转成文字，通过无线网络发给小车。小车上的 Agent 用 Rust 写的，收到文字调大模型生成回复，把文字原路传回手机。手机再把文字念出来。中间加了一台家里的电脑做消息转发，因为手机和小车不在同一个网络上，需要一个中转站。

手机端做了个原生 App，Swift 写的。界面很简单，聊天记录加一个输入框。有两种用法：打字发消息，或者点电话按钮直接语音对话。语音模式是个循环——手机听你说话，转成文字发给小车，小车想好回复传回来，手机念给你听，念完继续听，跟打电话一样。这个循环里最麻烦的是音频切换：听的时候要用麦克风，说的时候要用扬声器，两个不能同时开，每次切换都要正确释放再重新申请，不然就卡死。

小车端是整个系统的大脑。收到文字后调大模型推理，大模型可以调用工具——控制电机前后左右、读传感器数据、执行命令行、读写文件、甚至写入长期记忆。整个 Agent 循环是自动的，大模型自己决定要不要调工具、调几次，直到认为回答完整了才把最终结果发回来。

从"0 硬件知识买了个小车底盘"到现在三层架构跑通，这个系统终于能用了。接下来给小车装超声波传感器，让它能看见路，不再是个瞎子。
